Compiler toolchain and TPLs for ICED

Searching for appropriate ICED TPL module

Start by typing module avail

If you do not see the modules on /Projects among the list, do the following:

  • Module files are stored for all users at /Projects/IR/Users/admin/moduleFiles
  • The workstations can be told to uses these files by modifying the hidden file .modulespath found in /usr/share/Modules/init/
  • Use sudo to modify the file and add the following line to the bottom: /Projects/IR/Users/admin/moduleFiles/

Load the Appropriate Module

Choose from: module load TPL/ICED or module load SL7/TPL/ICED

What the Module Does

  • Selects the appropriate MPICH compiler
    • Version on /Projects for SL6
    • System version for SL7
  • Sets the CC, CXX, and FC environment variables
  • Sets the CMAKE_PREFIX_PATH to point to the IRAD, IMPACT, and Trilinos TPLs that are required for ICED

Building ICED out-of-source

  1. Create a separate build directory
  2. cmake /path/to/ICED/source
  3. make
  4. make install (optional to install elsewhere)
  5. make docs_iced (optional to build doxygen)

Notes

  • You only have to use -DCMAKE_INSTALL_PREFIX if you want to install the executables and libraries in some other location
  • It is fine to compile and run from the build directory
  • If you want to view and/or edit the Doxygen, be sure to include the CMake flag -DBUILD_DOCUMENTATION=ON (the default is OFF)
