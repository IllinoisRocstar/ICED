Building ICED on Mac

ICED: An Integrated Computational System for Electrochemical Device Design and Simulation.

The base system with simple modules will be released cost-free to all government, industry, and academic users to generate interest in the system. Licensable modules based on advanced simulation techniques and target applications will be licensed to industrial users for a fee and to academic institutions for a reduced fee.

These instructions pertain to building ICED on the Linux platforms. As of now, it is unknown as to whether ICED will be available on a Windows-based machine.
ICED has three third-party library requirements: IMPACT, IRAD, and Trilinos.

Note
The instructions presented herein require a Unix terminal. As such the commands are fairly general, but the default shell on Mac is bash, and the following commands therefore use bash syntax. Users should set path variables using the form dictated by their shell environment. The general forms for bash and csh are given here for ease-of-use. Note the syntax; it is very important! Users can determine their current shell environment by using SHELL.

echo $SHELL
/bin/bash: export USR_VAR="/path/to/user/location/"
../bin/csh: setenv USR_VAR /path/to/user/location

Installation Prerequisites

  • Obtain Xcode and the associated command-line tools

    Applications > Utilities > Terminal
    xcode-select - -install

  • Install gcc and gfortran from following links:

    gcc-5.1-bin.tar.gz and gfortran-5.1-bin.tar.gz

  • Install the gcc and gfortran binaries

    cd ~/Downloads
    gunzip gcc-5.1-bin.tar.gz
    sudo tar -xvf gcc-5.1-bin.tar -C /
    gunzip gfortran-5.1-bin.tar.gz
    sudo tar -xvf gfortran-5.1-bin.tar -C /

  • Obtain CMake from following link:

    cmake-3.4.0-Darwin-x86_64.dmg

  • Alias the various CMake commands for later use

    alias cmake="/Applications/CMake.app/Contents/bin/cmake"
    alias ctest="/Applications/CMake.app/Contents/bin/ctest"
    alias ccmake="/Applications/CMake.app/Contents/bin/ccmake"

  • Obtain OpenMPI from following link:

    openmpi-1.10.1.tar.gz

  • Build OpenMPI (Note: these instructions are copied from the 'INSTALL' file in the OpenMPI source for convenience)

    cd ~/Downloads
    tar xf openmpi-X.Y.Z.tar.gz
    cd openmpi-X.Y.Z
    mkdir build
    cd build
    ../configure - -prefix=/where/to/install
    [...a lot of output...]
    make all install

  • Add the paths to OpenMPI bin and lib to the environment settings

    export PATH=/path/to/your/OpenMPI/lib:$PATH
    export PATH=/path/to/your/OpenMPI/bin:$PATH

  • IMPACT-specific requirements
    • Obtain HDF4 from following link:

      hdf-4.2.5-mac-intel-x86_64.tar.gz

    • Obtain SZIP and JPEG from following links:

      szip-2.1-MacOSX-intel.tar.gz and jpeg-6b-MacOSX-intel.tar.gz

    • Install the IMPACT-specific binaries

      cd ~/Downloads
      tar -xvzf jpeg-6b-MacOSX-intel.tar.gz.tgz -C /path/of/your/choice/for/JPEG
      tar -xvzf szip-2.1-MacOSX-intel.tar.gz -C /path/of/your/choice/for/SZIP
      tar -xvzf hdf-4.2.5-mac-intel-x86_64.tar.gz -C /path/of/your/choice/for/HDF

Building IMPACT and IRAD

Because the IMPACT and IRAD infrastructures are really just used as support libraries, the following build scenario is recommended. Note that these support libraries should be built out-of-s ource but installed in the same place. Note that MPI and HDF4 are required.

export SZIP_PATH=/full/path/to/SZIP binaries
export JPEG_PATH=/full/path/to/JPEG binaries
export HDF4_PATH=/full/path/to/HDF4 binaries
cd TPLs/
git clone https://github.com/IllinoisRocstar/IMPACT IMPACT
cd IMPACT
git clone https://github.com/IllinoisRocstar/IRAD IRAD
cd ../
mkdir Build_TPLs Install_TPLs
cd Build_TPLs
mkdir IMPACT-IRAD
cd IMPACT-IRAD
cmake ../../IMPACT -DCMAKE_INSTALL_PREFIX=../../Install_TPLs -DCMAKE_PREFIX_PATH =$SZIP_PATH\;$JPEG_PATH\;$HDF4P_PATH
make
export DYLD_LIBRARY_PATH=/full/path/to/IMPACT/install/lib:$DYLD_LIBRARY_PATH
make test
make install

Building Trilinos

The Trilinos Project from Sandia National Laboratory is an effort to develop algorithms and enabling technologies within an object-oriented software framework for the solution of large-scale, complex multi-physics engineering and scientific problems. A unique design feature of Trilinos is its focus on packages. More information on Trilinos can be found here: https://trilinos.org/.

Because the Trilinos is another support library, the following build scenario is recommended. Note that the Trilinos support library should be built out-of-source but installed in the same place as the other support libraries. Trilinos is also available for download as a tarball, but we utilize Sandia's public git repository for simplicity. Note that use of the git version requires an additional argument to be passed to CMake when building from the Trilinos "public repository" version: -DTrilinos_ASSERT_MISSING_PACKAGES=OFF. For tarball of the current or other versions of Trilinos, please see their website: https://trilinos.org/download/.

Trilinos takes a long time to build (over an hour). Grap some coffee and check back later!

cd ../../
git clone https://github.com/live-clones/trilinos.git Trilinos
cd Build_TPLs
mkdir Trilinos
cd Trilinos
cmake ../../Trilinos -DCMAKE_INSTALL_PREFIX=../../Install_TPLs \
-DCMAKE_CXX_FLAGS:STRING="-w" \
-DTrilinos_ASSERT_MISSING_PACKAGES=OFF \
-DTrilinos_ENABLE_CXX11=OFF \
-DBUILD_SHARED_LIBS:BOOL=ON \
-DTPL_ENABLE_MPI=ON \
-DCMAKE_BUILD_TYPE=Release \
-DTrilinos_ENABLE_Epetra:BOOL=ON \
-DTrilinos_ENABLE_Teuchos:BOOL=ON \
-DTrilinos_ENABLE_Thyra:BOOL=ON \
-DTrilinos_ENABLE_Rythmos:BOOL=ON \
-DTrilinos_ENABLE_NOX:BOOL=ON \
-DTrilinos_ENABLE_TESTS=ON
make
make install

A script is provided in the configuration directory in the ICED source as an alternative to this rather convoluted command line input. It can be executed from the desired build directory as

do-configure_Trilinos ../../Trilinos -DCMAKE_INSTALL_PREFIX=../../Install_TPLs

This is a handy way to save a particular software configuration for future builds or refernece.

Building ICED

ICED is a multiphysics, multiscale, multifidelity simulation suite for advanced battery modeling. Eventually the base ICED system wil be open-source and available on GitHub, but for now, we assume that a tarball is the main distribution strategy.

cd ~/ICED_Suite
tar -xvzf ICED-0.tgz
mkdir Build Install
cd Build
cmake ../ICED-0 -DCMAKE_PREFIX_PATH=/full/path/to/TPLs/Install_TPLs/ -DCMAKE_INSTALL_PREFIX=../Install
make
export DYLD_LIBRARY_PATH=/full/path/to/Trilinos/install/lib\;/full/path/to/ICED/install/lib:$DYLD_LIBRARY_PATH
make test
make install
