Constructor.

Build a DeviceComponent from an existing SolutionData

Each DeviceComponent has it's own SolutionData object. We copy the data from the device level and then read the input file to overwrite the data if it is redefined.

Parameters
inputNames- A vector with two strings. The first names the object and the second provides an input file to be read.
solnData- A SolutionData object
globin- A GlobalType object
Returns
Pointer to a DeviceComponent.
